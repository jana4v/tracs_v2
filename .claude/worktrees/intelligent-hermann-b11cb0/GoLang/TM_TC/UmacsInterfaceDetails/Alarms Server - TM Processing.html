    Alarms Server¶

    Request¶

    • Websocket Address: ws://<WEB_SCHEMACS_SERVER_IP_ADDRESS>:9072

    To Subscribe for Alarms, following message should be sent to the websocket server:

    {
    "msg_type": "alarms",
    "stn_id": "ANY",
    "tm_id": "TM1"
}
    • msg_type: should be alarms
    • stn_id: Station Identifier (Ex: ANY/GCO1/GCO2/GCO3...)
    • tm_id: Station Identifier (Ex: TM1/TM2/ANY)

    *Both stn_id and tm_id cannot be ANY

    Response¶

    Once a valid request is received, alarms info messages will be sent as reponse continuously until the websocket connection is closed.

    Following is the response structure for each Pid with Alarm Info:

    {
    "pid": "JPLA0003",
    "alarm_type": 3,
    "value": "95.0",
    "warning_ll": -90.0,
    "warning_ul": 90.0,
    "danger_ll": -100.0,
    "danger_ul": 100.0,
    "initial_value": "",
    "warning_state_value": "",
    "danger_state_value": "",
    "frozen_duration": 0,
    "alarm_msg": "JPLA0003 Crossed Limits",
    "alert_flag": 2,
    "frame_timestamp": "28-DEC-2023 15:20:56.452"
}


    • pid:CDB PID
    • alarm_type: Alarm Type defined in alarms template
      • ALARM_TYPE_HIGH = 1
      • ALARM_TYPE_LOW = 2
      • ALARM_TYPE_EXCLUSIVE = 3
      • ALARM_TYPE_INCLUSIVE = 4
      • ALARM_TYPE_CHANGE = 5
      • ALARM_TYPE_STATE = 6
      • ALARM_TYPE_FROZEN = 7
    • value: Processed value in string
    • warning_ll: Warning Lower Limit
    • warning_ul: Warning Upper Limit
    • danger_ll: Danger Lower Limit
    • danger_ul: Danger Upper Limit
    • initial_value: Initial State Value
    • warning_state_value: Warning State Value
    • danger_state_value: Danger State Value
    • frozen_duration: Number of Master frames monitored to declare as Frozen value
    • alarm_msg: Alarm Message
    • alert_flag: Alert Flag
      • Data Break: -1
      • Normal: 0
      • Danger: 1
      • Warning: 2
    • frame_timestamp: Frame timestamp at which the pid is evaluated for any Alarm
