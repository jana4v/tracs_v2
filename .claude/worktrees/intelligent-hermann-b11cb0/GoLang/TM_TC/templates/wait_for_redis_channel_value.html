Wait for Redis Channel Value Endpoint Documentation

POST /wait_for_redis_channel_value

Subscribes to a Redis channel and waits for a specific command value. If the command is received within the timeout period, it responds with the command. Otherwise, it times out.

Request

        Headers:
            Content-Type: application/json
        Body:
        {
            "value": "exampleCommand",
            "channel_name": "exampleChannel",
            "timeout": 30
        }

Response

        Headers:
            Content-Type: text/plain
        Body:
            Received command: exampleCommand

Example Request

        curl -X POST \
        http://localhost:8080/wait_for_redis_channel_value \
        -H 'Content-Type: application/json' \
        -d '{
              "value": "exampleCommand",
              "channel_name": "exampleChannel",
              "timeout": 30
            }'

Example Response

        Received command: exampleCommand

Error Responses

        Status Code: 400 Bad Request
        Body:
        {
            "error": "Invalid request payload"
        }

        Status Code: 400 Bad Request
        Body:
        {
            "error": "Command and ChannelName fields are required"
        }
        
        Status Code: 408 Request Timeout
        Body:
        {
            "error": "Timeout waiting for command"
        }
