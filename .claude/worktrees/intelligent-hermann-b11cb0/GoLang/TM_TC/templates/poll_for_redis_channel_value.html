Poll for Redis Channel Value Endpoint Documentation

POST /poll_for_redis_channel_value

Polls a Redis channel for a specific command value. If the command is received within a specified period, it responds with the command status. Otherwise, it times out.

Request

        Headers:
            Content-Type: application/json
        Body:
        {
            "uuid": "unique-request-id",
            "value": "exampleCommand",
            "channel_name": "exampleChannel"
        }

Response

        Headers:
            Content-Type: application/json
        Body:
        {
            "status": true,
            "uuid": "unique-request-id"
        }

Example Request

        curl -X POST \
        http://localhost:8080/poll_for_redis_channel_value \
        -H 'Content-Type: application/json' \
        -d '{
              "uuid": "unique-request-id",
              "value": "exampleCommand",
              "channel_name": "exampleChannel"
            }'

Example Response

        {
            "status": true,
            "uuid": "unique-request-id"
        }

Error Responses

        Status Code: 400 Bad Request
        Body:
        {
            "error": "Invalid request payload"
        }

        Status Code: 400 Bad Request
        Body:
        {
            "error": "Command, ChannelName, and UUID fields are required"
        }
        
        Status Code: 408 Request Timeout
        Body:
        {
            "error": "Timeout waiting for command"
        }
