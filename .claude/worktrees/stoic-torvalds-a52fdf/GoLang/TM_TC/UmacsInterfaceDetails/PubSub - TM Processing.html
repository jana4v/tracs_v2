    Real-Time Telemetry Data Access from SCHEMACS/UMACS for any Spacecraft Mission¶

    Introduction¶

    This software is designed to facilitate real-time access to processed telemetry data for various software systems. Few to name are SCADH, STK Interface (ISTRAC). It efficiently distributes the processed telemetry data in real-time over websockets, requiring client systems to establish subscribers to access the data. The communication protocol remains consistent for SCHEMACS & UMACS software versions. Additionally, this software can be made available at operational-centres/Test-Beds/MAC. The software's flexibility also allows for customization to meet any future specific needs of different applications. This software leverages websockets for real-time data transfer, ensuring compatibility with a variety of programming languages including Matlab, Javascript, Java, Python, Go, Rust etc. thus offering flexibility and ease of integration across different development environments. This data distribution architecture is based on Publisher-Subscriber design pattern.

    Architecture¶

    Normal or HK Telemetry Data Access¶

    • Websocket Address: ws://<SCHEMACS_SERVER_IP_ADDRESS>:9000

    To subscribe or unsubscribe any Telemetry Parameters, the following message should be sent to the websocket server:

    {
    "user_id": "STK_01",
    "msg_type": "ntm",
    "msg_payload": {
        "sc_id": "ADT-L1",
        "stream": "TM1-ANY",
        "action": "subscribe",
        "parameters": ["PWRS0090", "DTG10001"]
    }
}
    • user_id: User Identifier
    • msg_type: This field should always be "ntm"
    • sc_id: Spacecraft Identifier
    • stream: Represents a combination of Telemetry-Id and Station-Id. The format is "TMx-STN", where TMx is the Telemetry-Id (TM1, TM2) and STN is the Station-Id. "ANY" can be used as a placeholder to indicate no specific preference. For example, "TM1-ANY" implies Telemetry-Id as TM1 with any station, whereas "ANY-ANY" implies no preference for either telemetry or station Id. To target a specific telemetry and station, replace ANY with the desired identifiers, like "TM1-BLR" for Telemetry-Id TM1 and Station-Id BLR
    • action: subscribe (or) unsubscribe
    • parameters: List of parameters for subscribing (or) unsubscribing. To subscribe or unsubscribe all available telemetry parameters, add * in the parameters field as "parameters":["*"]

    *Separate requests have to be sent for subscribing to telemetry of other streams/type and Spacecraft

    All subscribed parameters information will come in response as a single JSON packet at frame interval specific to the subscribed Spacecraft. The details of the packet are as following:

    {
    "user_id": "STK_01",
    "msg_type": "ntm",
    "msg_payload": {
        "sc_id": "ADT-L1",
        "stream": "TM1-BLR",
        "frame_time": "11-DEC-2023 08:42:05:633",
        "error": "",
        "parameters_info": [
            {
                "param": "PWRS0090",
                "tm_cnt": 219,
                "float_v": 36.57598,
                "str_v": "36.58"
            },
            {
                "param": "DTG10001",
                "tm_cnt": 0,
                "float_v": 0,
                "str_v": "OFF"
            }
        ]
    }
}
    • user_id: User Identifier
    • msg_type: packet identifier field. This field will be "ntm"
    • sc_id: Spacecraft Identifier
    • stream: Represents a combination of Telemetry-Id and Station-Id. In the Request packet, the "stream" field can contain ANY as a placeholder for either Telemetry-Id or Station-Id. However, in the Response packet, ANY is replaced with the actual Telemetry-Id or Station-Id. The Response packet will always contain specific identifiers, and ANY will not appear in the reponse's "stream" field, thus ensuring precise and relevant information based on the request's parameters.
    • frame_time: Frame Timestamp
    • error: Error Information. If error field is not blank, then list of objects in parameters_info will be empty
    • parameters_info: List of JSON objects, each containing the subscribed parameter's telemetry data
    • param: CDB parameter
    • tm_cnt: Raw telemetry count of the parameter
    • float_v: Processed value of the parameter in float format
    • str_v: Processed value of the parameter in string format
