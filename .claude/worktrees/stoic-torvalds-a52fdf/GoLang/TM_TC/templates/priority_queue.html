Priority Queue Interface Documentation

Endpoints

  • POST /enqueue/{queueName}
  • GET /dequeue/{queueName}
  • POST /clear/{queueName}

POST /enqueue/{queueName}

Adds an item to the specified priority queue.

Request

        Headers:
            Content-Type: application/json
        Body:
        {
            "priority": 1,
            "value": "exampleValue"
        }

Response

        Headers:
            Content-Type: text/plain
        Body:
            Created

Example Request

        curl -X POST \
        http://localhost:8080/enqueue/exampleQueue \
        -H 'Content-Type: application/json' \
        -d '{
              "priority": 1,
              "value": "exampleValue"
            }'

Example Response

        Created

GET /dequeue/{queueName}

Removes and returns the item with the highest priority from the specified queue.

Request

        No request body required.

Response

        Headers:
            Content-Type: application/json
        Body:
        {
            "priority": 1,
            "value": "exampleValue"
        }

Example Request

        curl -X GET \
        http://localhost:8080/dequeue/exampleQueue

Example Response

        {
            "priority": 1,
            "value": "exampleValue"
        }

POST /clear/{queueName}

Clears all items from the specified queue.

Request

        No request body required.

Response

        Headers:
            Content-Type: text/plain
        Body:
            OK

Example Request

        curl -X POST \
        http://localhost:8080/clear/exampleQueue

Example Response

        OK

Error Responses

        Status Code: 400 Bad Request
        Body:
        {
            "error": "Invalid request payload"
        }

        Status Code: 404 Not Found
        Body:
        {
            "error": "Queue is empty or does not exist"
        }
